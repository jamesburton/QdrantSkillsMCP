<!-- Agent config format matrix from research -->

| Agent | Format | Root Key | User Path | Project Path |
|-------|--------|----------|-----------|--------------|
| Claude Code | JSON | mcpServers | ~/.claude.json | .mcp.json |
| Claude Desktop | JSON | mcpServers | %APPDATA%/Claude/claude_desktop_config.json (Win) | N/A |
| VS Code / Copilot | JSON | servers | (user mcp.json via VS Code) | .vscode/mcp.json |
| Copilot CLI | JSON | mcpServers | ~/.copilot/mcp-config.json | N/A |
| OpenAI Codex | TOML | [mcp_servers.name] | ~/.codex/config.toml | .codex/config.toml |
| opencode | JSON | mcp | ~/.config/opencode/opencode.json | opencode.json |
| kilocode | JSON | mcpServers | ~/.kilocode/mcp_settings.json | .kilocode/mcp.json |
| factory-droid | JSON | mcpServers | ~/.factory/mcp.json | .factory/mcp.json |

MCP server entry for QdrantSkillsMCP:
```json
{ "command": "dnx", "args": ["qdrant-skills-mcp"] }
```

Key differences:
1. VS Code uses "servers" not "mcpServers"
2. opencode uses "mcp" as root key and "command" is an array
3. Codex uses TOML format
4. Some agents need "type": "stdio" or "type": "local"
</interfaces>
