  <action>
    1. **SkillCrudTools.cs** -- `[McpServerToolType]` class with DI-injected services:
       - Constructor: `ISkillRepository`, `IEmbeddingService`, `ISessionTracker`, `SkillParser`, `ILogger<SkillCrudTools>`.

       - `[McpServerTool(Name = "add-skill")]` `AddSkill(string name, string content, bool overwrite = false, CancellationToken ct)`:
         - Validate name with SkillNameValidator. Return error string if invalid.
         - Parse content with SkillParser to extract metadata + body.
         - Override parsed name with the provided `name` parameter (name param is authoritative).
         - Generate embedding from description + body (concatenate for richer vector).
         - Call `repository.AddAsync(skill, embedding, overwrite, ct)`.
         - Return success message with skill name.
         - Mark as `Destructive = true, ReadOnly = false`.

       - `[McpServerTool(Name = "update-skill")]` `UpdateSkill(string name, string content, CancellationToken ct)`:
         - Validate name. Parse content. Generate new embedding.
         - Call `repository.UpdateAsync(skill, embedding, ct)`.
         - Return success message.
         - Mark as `Destructive = true, ReadOnly = false`.

       - `[McpServerTool(Name = "delete-skill")]` `DeleteSkill(string name, CancellationToken ct)`:
         - Validate name.
         - Call `repository.DeleteAsync(name, ct)`.
         - Return confirmation message.
         - Mark as `Destructive = true, ReadOnly = false`.

       - `[McpServerTool(Name = "archive-skill")]` `ArchiveSkill(string name, CancellationToken ct)`:
         - Validate name.
         - Call `repository.ArchiveAsync(name, ct)`.
         - Return confirmation message (skill hidden from search, restorable).
         - Mark as `Destructive = true, ReadOnly = false`.

    2. **SkillSearchTools.cs** -- `[McpServerToolType]` class:
       - Constructor: `ISkillRepository`, `IEmbeddingService`, `ISessionTracker`, `ILogger<SkillSearchTools>`.

       - `[McpServerTool(Name = "search-skills")]` `SearchSkills(string query, int maxResults = 5, float? temperature = null, bool includeContent = true, CancellationToken ct)`:
         - Generate embedding from query.
         - Map temperature to score_threshold: temperature is 0.0 (strict) to 1.0 (loose). Convert: `scoreThreshold = 1.0f - temperature` if temperature provided, else null (no threshold).
         - Call `repository.SearchAsync(embedding, maxResults, scoreThreshold, ct)`.
         - Build response:
           - Get already-loaded skills from session tracker.
           - Prefix: `"ALREADY LOADED SKILLS: skill-a, skill-b\n\n"` (empty string if none loaded). Per CONTEXT.md locked decision: both structured and text prefix.
           - For each result: if `includeContent`, return full skill (name, description, score, raw_content). If not, return metadata only (name, description, tags, score).
           - Only mark skills as loaded in session if `includeContent == true`. Per CONTEXT.md decision.
         - Return as JSON string (agents parse JSON).
         - Mark as `ReadOnly = true`.

       - `[McpServerTool(Name = "load-skill")]` `LoadSkill(string name, CancellationToken ct)`:
         - Per SRCH-04/SRCH-05: retrieve by name, always returns current version (no cache).
         - Validate name.
         - Call `repository.GetByNameAsync(name, ct)`.
         - If not found, return error message.
         - Mark as loaded in session tracker.
         - Return full skill content (raw_content for lossless retrieval).
         - Mark as `ReadOnly = true`.

       - `[McpServerTool(Name = "list-skills")]` `ListSkills(CancellationToken ct)`:
         - Call `repository.ListAsync(ct)`.
         - Return list of skill metadata (name, description, tags, updated_at).
         - Mark as `ReadOnly = true`.

    All tool methods return `string` (or `Task<string>`). Use `System.Text.Json.JsonSerializer.Serialize` for structured responses.
    All tool methods must have `[Description("...")]` attributes with clear descriptions for agent discoverability (MCP-02).
    Error handling: catch exceptions, log to ILogger (goes to stderr), return user-friendly error strings -- never throw from tool methods.

  <action>
    Create the application entry point in `src/QdrantSkillsMCP.Infrastructure/Program.cs`:

    ```csharp
    var builder = Host.CreateApplicationBuilder(args);

    // CRITICAL: ALL logging to stderr. Stdout is reserved for MCP JSON-RPC transport.
    // This is the #1 risk identified in research -- stdout pollution breaks MCP.
    builder.Logging.AddConsole(options =>
    {
        options.LogToStandardErrorThreshold = LogLevel.Trace;
    });

    // Configuration: supports appsettings.json, environment variables (QDRANT_SKILLS__ prefix),
    // and command-line args. Standard .NET precedence: CLI > env > file.
    // Also support qdrant-skills.json as alternative config file name.
    builder.Configuration.AddJsonFile("qdrant-skills.json", optional: true, reloadOnChange: false);

    // Register all infrastructure services
    builder.Services.AddQdrantSkillsInfrastructure(builder.Configuration);

    // MCP server with stdio transport -- tools auto-discovered from assembly
    builder.Services
        .AddMcpServer()
        .WithStdioServerTransport()
        .WithToolsFromAssembly();

    await builder.Build().RunAsync();
    ```

    Key points:
    - `LogToStandardErrorThreshold = LogLevel.Trace` forces ALL log levels to stderr.
    - `AddJsonFile("qdrant-skills.json")` adds the alternative config file per CONTEXT.md decision.
    - `WithToolsFromAssembly()` discovers SkillCrudTools and SkillSearchTools automatically (MCP-02).
    - `WithStdioServerTransport()` enables stdio MCP transport (MCP-01).
    - Environment variable prefix `QDRANT_SKILLS__` is handled automatically by .NET configuration binding (double-underscore is the standard section separator).
    - No `Console.WriteLine` anywhere -- enforce via code review.

    Also verify the Infrastructure project's csproj has `<OutputType>Exe</OutputType>` to produce a runnable binary.
