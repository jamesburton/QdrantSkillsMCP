<objective>
Build the `--setup` wizard that detects installed AI agents and writes MCP server configuration entries into their config files, with backup and fallback to printed snippets. After writing MCP config, also writes SKILL.md to the agent's skill directory.

Purpose: Users can configure QdrantSkillsMCP for any supported agent in one command, removing the manual JSON/TOML editing barrier. Agents also receive the bundled SKILL.md for immediate skill awareness.
Output: Working setup wizard supporting Claude, Copilot, Codex, opencode, kilocode, factory-droid, and other agents with config auto-writing, SKILL.md placement, and snippet fallback.
</objective>

| Agent | Format | Root Key | User Path | Project Path | Skill Directory |
|-------|--------|----------|-----------|--------------|-----------------|
| Claude Code | JSON | mcpServers | ~/.claude.json | .mcp.json | ~/.claude/skills/qdrant-skills-mcp/ |
| Claude Desktop | JSON | mcpServers | %APPDATA%/Claude/claude_desktop_config.json (Win) | N/A | N/A (no skill directory) |
| VS Code / Copilot | JSON | servers | (user mcp.json via VS Code) | .vscode/mcp.json | N/A |
| Copilot CLI | JSON | mcpServers | ~/.copilot/mcp-config.json | N/A | N/A |
| OpenAI Codex | TOML | [mcp_servers.name] | ~/.codex/config.toml | .codex/config.toml | N/A |
| opencode | JSON | mcp | ~/.config/opencode/opencode.json | opencode.json | N/A |
| kilocode | JSON | mcpServers | ~/.kilocode/mcp_settings.json | .kilocode/mcp.json | N/A |
| factory-droid | JSON | mcpServers | ~/.factory/mcp.json | .factory/mcp.json | N/A |

MCP server entry for QdrantSkillsMCP:
```json
{ "command": "dnx", "args": ["qdrant-skills-mcp"] }
```

Key differences:
1. VS Code uses "servers" not "mcpServers"
2. opencode uses "mcp" as root key and "command" is an array
3. Codex uses TOML format
4. Some agents need "type": "stdio" or "type": "local"
5. Only Claude Code has a skill directory for SKILL.md placement; other agents rely on the get-skill-guide MCP tool
