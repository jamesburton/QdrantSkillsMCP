  <behavior>
    - DetectShell returns PowerShell when PSModulePath env var is set
    - DetectShell returns Bash when SHELL ends with /bash
    - DetectShell returns Zsh when SHELL ends with /zsh
    - DetectShell returns Cmd on Windows when no PowerShell/SHELL indicators
    - DetectShell returns Bash as ultimate fallback
    - GenerateEnvTemplate(ShellType.Bash, configValues) produces "export QDRANT_SKILLS__Key=value" lines
    - GenerateEnvTemplate(ShellType.PowerShell, ...) produces "$env:QDRANT_SKILLS__Key = value" lines
    - GenerateEnvTemplate(ShellType.Cmd, ...) produces "set QDRANT_SKILLS__Key=value" lines
    - Unset values are commented out in the template
    - Template covers all configurable keys from ConfigManager
  </behavior>
  <action>
    Create ShellDetector as a static class with ShellType enum (Bash, Zsh, PowerShell, Cmd).

    DetectShell(): Check PSModulePath (PowerShell), then SHELL env var (bash/zsh), then OperatingSystem.IsWindows() for Cmd, fallback Bash.

    GenerateEnvTemplate(ShellType shell, Dictionary<string, string?> currentValues): For each configurable key from ConfigManager, emit the appropriate format. Keys with non-null values are uncommented; null/empty values are commented. Use ConfigManager's static list of configurable keys.

    For testability, add an overload DetectShell(Func<string, string?> getEnvVar, bool isWindows) so tests don't depend on actual environment.

    Format examples from research:
    - Bash/Zsh: `export QDRANT_SKILLS__QdrantHost="localhost"` or `# export QDRANT_SKILLS__QdrantApiKey=""`
    - PowerShell: `$env:QDRANT_SKILLS__QdrantHost = "localhost"` or `# $env:QDRANT_SKILLS__QdrantApiKey = ""`
    - Cmd: `set QDRANT_SKILLS__QdrantHost=localhost` or `REM set QDRANT_SKILLS__QdrantApiKey=`
  </action>
