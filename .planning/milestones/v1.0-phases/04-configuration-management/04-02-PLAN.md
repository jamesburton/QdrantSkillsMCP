    Create ConfigCommand as a static class in Cli/Commands/ namespace (same pattern as other commands in that directory).

    public static async Task<int> RunAsync(ConfigManager configManager, string[] args)

    - "show": Call configManager.GetAllWithSources(), format as table using Spectre.Console Table with columns [Key, Value, Source]. Mask secrets via SecretMask unless --reveal flag present. Use AnsiConsole.Create() with AnsiConsoleOutput(Console.Out) per established pattern (03-01 decision).

    - "use": Call configManager.UseProfileAsync with next arg.
    - "env": Call ShellDetector.DetectShell(), get current values from configManager, call ShellDetector.GenerateEnvTemplate, print result.
    - "validate": See Task 2.
    - No subcommand (just "--config"): Enter interactive wizard (see below).
    - Unknown: Print usage, return 1.

    For tests: ConfigCommand takes ConfigManager (with temp dirs) so all file operations are testable. Use StringWriter to capture Console.Out where needed.
